# DeltaTest 项目技术栈设计

基于《整体架构重梳理：双模式驱动的Web自动化测试平台》架构文档，为 Vue 前端、Java 后端、C 计算引擎、Python AI 服务四条技术线制定统一技术选型，确保各层独立演进、接口标准化。

---

## 一、全局技术约束

| 约束项 | 规范 |
|--------|------|
| JDK 版本 | **21 LTS**（虚拟线程 + Record 模式匹配） |
| Node.js 版本 | **20 LTS** |
| Python 版本 | **3.12** |
| C 标准 | **C17**（兼容 GCC 12+ / MSVC 19.3+） |
| 编码 | 全项目 UTF-8 |
| API 规范 | RESTful + OpenAPI 3.1 |
| 通信协议 | HTTP/1.1（同步）、WebSocket（实时推送）、gRPC（C 引擎高性能调用）、AMQP（异步任务分发） |
| 容器化 | Docker + Kubernetes，所有服务容器化交付 |
| 版本管理 | Git + 语义化版本（SemVer 2.0） |

### 5.4 AI 服务核心功能模块

| 模块 | 职责 | LLM 用途 |
|------|------|----------|
| **风险评估模块** | 接收变更范围，评估风险等级（high/medium/low） | Prompt → 结构化 JSON 输出 |
| **变更摘要模块** | 生成变更说明与测试建议 | Prompt + 上下文 → 自然语言摘要 |
| **用例生成模块** | 根据需求/页面描述自动生成测试步骤与断言 | Prompt + 页面结构 → 结构化步骤 JSON |
| **根因分析模块** | 分析失败用例的可能原因与修复建议 | Prompt + 执行日志 + 错误信息 → 原因 + 建议 |
| **语义匹配模块** | 变更影响范围与已有用例的语义匹配 | Embedding + Milvus 相似度检索 |

### 5.5 智能体（Agent）架构

> 详细规范参见 [agent.md](../.trae/skills/项目规范/agent.md)

| 类别 | 技术选型 | 版本 | 选型理由 |
|------|----------|------|----------|
| Agent 架构 | **Master-Slave 协作模式** | - | Master 统一意图路由与结果聚合，Sub-Agent 封装单一 AI 能力 |
| 工具注册 | **装饰器 + ToolRegistry** | - | 声明式工具定义，应用启动自动扫描注册 |
| 短期记忆 | **Redis** | 7.x | 会话上下文保持，TTL 自动过期 |
| 长期记忆 | **MySQL** → **Milvus** | - | Phase 3 初期 MySQL 结构化查询，后续迁移向量检索 |
| Agent 通信 | **进程内 async call** | - | 同一 FastAPI 进程，无额外 RPC 开销 |

**Agent 能力矩阵**：

| Agent | 角色 | LLM 模型 | Prompt 策略 |
|-------|------|---------|------------|
| MasterAgent | 意图路由 + 编排 + 聚合 | gpt-4o | 分类路由 |
| RiskAnalysisAgent | 风险等级评估 | gpt-4o | JSON Schema 约束 + few-shot |
| ChangeSummaryAgent | 变更摘要与测试建议 | gpt-4o-mini | Chain-of-Thought |
| RootCauseAgent | 失败根因分析 | gpt-4o | 多轮分析 + 置信度自评 |
| CaseGenerationAgent | 测试用例自动生成 | gpt-4o | 分步生成 + 页面元素注入 |
| SemanticMatchAgent | 语义匹配关联用例 | 不使用 LLM | Embedding 向量检索 |

## 六、跨语言接口规范

### 6.1 接口矩阵

| 调用方 | 被调用方 | 协议 | 序列化 | 超时策略 |
|--------|----------|------|--------|----------|
| Vue → Java | HTTP REST / WebSocket | JSON | 请求 30s / WebSocket 长连接 |
| Java → Python | HTTP REST | JSON | 连接 5s / 读取 60s（AI 耗时较长） |
| Java → C | gRPC | Protobuf | 连接 3s / 读取 30s |
| Java → Playwright | RabbitMQ AMQP | JSON | 异步，无超时（消费者自行处理） |
| Playwright → Java | HTTP 回调 + AMQP | JSON | 回调 10s / AMQP 异步 |

### 6.2 统一响应格式（HTTP）

```json
{
  "code": 200,
  "message": "success",
  "data": { },
  "timestamp": 1719936000000
}
```

### 6.3 统一错误码规范

| 错误码范围 | 含义 | 示例 |
|-----------|------|------|
| 200 | 成功 | - |
| 400 | 客户端参数错误 | 缺少必填字段 |
| 401 | 未认证 | Token 过期 |
| 403 | 无权限 | RBAC 拦截 |
| 404 | 资源不存在 | 用例 ID 不存在 |
| 409 | 冲突 | 乐观锁版本冲突 |
| 500 | 服务端内部错误 | 未捕获异常 |
| 502 | 上游服务不可用 | AI 服务超时降级 |
| 503 | 服务暂不可用 | C 引擎异常 |
